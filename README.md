# RefillWatch — site repo

Astro static site for refillwatch.org. Content is fed by the
`amazon-affiliate-scout` sibling repo, augmented by human-edited pillar
pages and supporting articles drafted via the ChatGPT prompt library.

## Quick start

```bash
npm install
npm run dev          # http://localhost:4321
npm run build        # generates dist/
npm run preview
```

## Content model

Three Astro content collections in `src/content/`:

- **products/** — one file per Amazon review, generated by the scout's
  `python -m scout.cli generate` command. Human fills in the `<!-- SECTION -->`
  blocks using the prompts in [CHATGPT_PROMPTS.md](./CHATGPT_PROMPTS.md),
  then flips `status: published`.
- **pillars/** — long-form authority guides, hand-written. 2,000-3,000
  words each, one per major topic.
- **articles/** — supporting Q&A and how-to pieces that link back to pillars.

Schema is enforced by Zod in `src/content/config.ts`. A file with the
wrong frontmatter fields fails the build early with a clear error.

## Content workflow

1. **Data**: `python -m scout.cli scout --max 400` in the scout repo.
2. **Score**: already happens in `scout`. Verify with `python -m scout.cli show --top 20`.
3. **Export**: `python -m scout.cli generate --top 80` drops skeletons
   into `src/content/products/`.
4. **Draft**: for each skeleton, paste the file into ChatGPT with
   prompt #1 from CHATGPT_PROMPTS.md. Paste the response back.
5. **Edit**: 15 minutes per review. Voice, safety, specific claims.
6. **Publish**: change `status: draft` → `status: published`, commit.
7. **Deploy**: `git push origin main` — VPS auto-rebuilds.

## Pillar pages

Write the first one by hand. Use prompt #2 from CHATGPT_PROMPTS.md for
the rough structure, then edit heavily. Pillars should be the smartest
pages on the site — they earn the domain authority that carries the
product reviews.

## FTC / legal notes

Affiliate disclosure appears on every page via `<AffiliateDisclosure />`
in the base layout. Don't remove it. The dedicated `/disclosure` page
is the canonical long version.

Amazon's operating agreement requires the disclosure to be "clear and
conspicuous" — our above-the-fold banner plus the dedicated page satisfy
this. If you add a different affiliate program (Rakuten, Impact, etc.),
update the disclosure page text to list it.

## Tracking what matters

Pages indexed (Google Search Console), email subscribers, Amazon clicks.
Track these weekly. Ignore everything else for the first 90 days.
